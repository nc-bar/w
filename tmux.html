tmux

client-server system to multiplex terminals. client manages the display of terminals and server receives keystrokes from the clients.

panes are rectangular subsections within the terminal. the terminal and panes are related 1-to-1. the active pane is the one input from the user. a window is divided into one or more panes; a window spans an entire terminal. the user can choose one pane (terminal) to be the active one, as well as create, resize and delete them. every window has a name, and an index within a session; if a name is not set by the user the new window will have a default name. the user can choose the active window in the session. the client always displays the active window.

a session has a set of windows and is attached to a client. several sessions can connect to the server and windows can be assotiated with multiple sessions (that is, windows are not necesarilly exclusive to a session). the session can be detached from a client and later be attached again to the same or other clients. 

  • tmux
    launches the server and creates a client with a session attached, a window and a pan
  • kill-session -t session_name
    kills session named 
    session_name
  • kill-session -a
    kills all sessions except the current
  • kill-session -a -t session
    kills all sessions except session
  • ls
    lists all the sessions
  • new -s session -n window
    creates a new session of name session and a window with name window
  • info
    displays information about sessions, windows, panes, etc.

each client shows a status line on the bottom line of the screen:

[0] 0:ash- 1:ash 2:nano*      "localhost" 02:17 28-Feb-22

the 

[N]
at the left contains the session number; it's followed by a list of the windows in the session using the format
 index:name
; a 
*
marks the active windows. 

tmux
clients have a set of predefined functions. users can use them in the command prompt or key bindings. the command prompt appears in the bottom line of the active window, temporarily in place of the status bar, when activated by 
ctrl-b :
. to call a function, its name is typed into the prompt along with its arguments and then pressing enter. the status bar returns after the function ends.

each function can be invoked by a keybinding, of the form 

ctrl-b s
where 
ctrl-b
means pressing the control key and b, releasing them at the same time, and 
s
is a string of characters (for example 
ctrl-b c
).

some functions take some arguments from the command promp; when calling them through a key binding, the bottom line changes to the promp temporarily.

key bindings are prefixed by 

ctrl-b
by default. the commands starting with 
:
are used in the command prompt.

tmux has a copy mode which, among other things, can copy and scroll old text not shown in the terminal anymore. copy mode can be entered, if configured properly, by scrolling with the mouse. there's also a keybinding: 

cntrl-b [
enters copy mode.

tmux provides a hierarchical view of sessions, windows and panes. it's called tree mode and it displays a tree-like view of sessions, windows and panes. using the up and down arrow keys to select an element and the right and left arrow to expand or collapse it: sessions into windows and windows into panes. a window can be set as currentby pressing enter. 

  • s
    enter tree mode without expanding session 
  • t
    enter tree mode with expanded sessions and windows
pessing 
esc
exits any mode (tree, copy, etc). tmux reads the configuration file 
~/.tmux.conf
when the server starts. t contains a list of commands to be executed in order. a running server can execute another configuration file by: 

:source file

execute tmux comands in the file
